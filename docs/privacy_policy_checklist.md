# プライバシーポリシー（チェックリスト）

最終更新: 2025-12-21

このドキュメントは、ストア公開（App Store / Google Play）に向けて、Todo/継続目標アプリ（端末ローカル保存・同期なし）＋ **バナー広告（非パーソナライズ前提）** を導入する際に、プライバシーポリシーへ最低限記載すべき項目のチェックリストです。

※法的助言ではありません。実際の文面は、利用する広告SDK（例: AdMob等）や配信設定、対象地域（EU等）に合わせて調整してください。

---

## 1. 基本情報

- [ ] 事業者/開発者名
- [ ] 連絡先（メールアドレス）
- [ ] 制定日 / 改定日
- [ ] 適用範囲（本アプリ/本サービス）

## 2. アプリが扱うデータ（一次データ）

本アプリの仕様（同期なし・ローカル保存）に合わせ、次を明確化します。

- [ ] ユーザーが入力するデータ
  - [ ] Todoのタイトル/メモ
  - [ ] 継続目標の目標名/分母/分子
- [ ] 保存場所
  - [ ] 端末ローカル（クラウド送信・アカウント同期なし）
- [ ] アプリ削除時の扱い
  - [ ] 端末内データが削除される（復元/引き継ぎなし）

## 3. 第三者（広告SDK）によるデータ取り扱い

バナー広告の配信には通常、第三者SDKが関与します。アプリ自体が同期しなくても、広告配信のためにSDKが端末情報等を扱う可能性があるため、ここを明記します。

- [ ] 利用する第三者サービス名（例: Google AdMob 等）
- [ ] 収集・利用され得る情報のカテゴリ（例）
  - [ ] 端末情報（OS/端末モデル等）
  - [ ] 広告識別子（IDFA/AAID 等）※設定や同意状況により取得可否が変わる
  - [ ] IPアドレス等のネットワーク情報
  - [ ] 広告表示/クリック等の利用状況（計測）
- [ ] 利用目的
  - [ ] 広告配信、広告の効果測定、不正防止 等
- [ ] 共有先（第三者）
  - [ ] 広告配信事業者およびそのパートナー

## 4. パーソナライズ広告の扱い（MVP方針）

要件の通り、MVPでは非パーソナライズ広告を基本とする前提を明記します。

- [ ] 本アプリは原則として非パーソナライズ広告を表示する
- [ ] パーソナライズ広告を行う場合は、別途同意/設定を用意する（将来拡張）

## 5. 同意・オプトアウト（地域/OS依存）

MVPで「非パーソナライズ」を基本にしても、地域やSDK設定により同意が必要になる場合があります。現時点の方針を明記できるようにしておきます。

- [ ] EU等（GDPR等）対象の場合の同意取得方針（将来対応も含め記載）
- [ ] iOSのATT（AppTrackingTransparency）への対応方針（将来対応も含め記載）
- [ ] オプトアウト手段（OS設定、広告設定ページへの案内等）

## 6. データ保持期間

- [ ] アプリ内データ（Todo等）は端末内に保持される
- [ ] 第三者SDKが保持するログ等は当該第三者のポリシーに従う

## 7. セキュリティ

- [ ] 端末内データの取り扱い（一般的なセキュリティ対策を行う、等の範囲で記載）

## 8. 子どものプライバシー

- [ ] 子ども向け（児童向け）に該当する/しないの方針
- [ ] 該当する場合の追加要件（例: 年齢制限、同意、広告の取り扱い）

## 9. ポリシー変更

- [ ] 変更時の告知方法

## 10. 外部リンク

- [ ] 広告SDK（第三者）のプライバシーポリシーへのリンク
  - 例: Googleのポリシー、AdMob関連ページ 等

---

## ストア提出で別途必要になりやすいもの（メモ）

- [ ] Google Play: Data safety（データセーフティ）フォームの記入
- [ ] App Store: App Privacy（プライバシー）申告（Nutrition Labels）

このチェックリストは、実装（導入SDK）とストア申告の整合を取るための「抜け漏れ防止」用途として使ってください。
